# FrequencyOS — Control Layer / 控制层架构

> 三份文档合流为统一架构：
> - 《频域组织度重建手册》— 三频评分 / 90 天锁定 / 摇摆拦截
> - 《信息传递与控制》— 五层闭环 / 噪声滤波 / 可能性空间收缩
> - 《虚空建筑师全胜手册》— 心力账户 / 72h 微验证 / 不高考只赢

---

## 核心命题

> ProjectOS 不应该只是"记住老茅想过什么"，而应该进一步变成：
> **让老茅每天更少摇摆、更少内耗、更少被噪声劫持，并更稳定地推进主频目标的控制系统。**

信息的意义，不是保存；
信息的意义，是帮助系统**缩小可能性空间**，并最终形成**控制**。

## L1 · 采样层（Reality Sampler）

| 项 | 说明 |
|---|---|
| 核心问题 | 不要求老茅整理现实，系统替老茅采样现实 |
| 输入 | 飞书消息 / 语音想法 / 本地文件 / 项目目录 / Git 状态 / 文章 PDF / 会议纪要 / 截图录屏 / 情绪波动 / 项目停滞 / 新想法冲动 |
| 输出 | 带标签的信号记录（来源 / 项目 / 类型 / 是否缩小可能性空间 / 信源可信度 / 情绪噪声强度 / 是否需要行动） |
| 文档来源 | 《信息传递与控制》 |

**采样原则**：低频采样，高频压缩，只在关键节点提醒。
不预测未来，不制造幻觉，只做状态压缩。

**落地**：V0.1 = `daily_log.md` 手动采样；V0.3 = Hermes Desktop 语音/屏幕自动采样。

## L2 · 滤波层（Noise Filter）

| 项 | 说明 |
|---|---|
| 核心问题 | 区分三类干扰，让信号浮出来 |
| 输入 | L1 的带标签信号 |
| 输出 | 三类分流：噪声池 / 制度接口 / 冷却池 |
| 文档来源 | 《信息传递与控制》 |

### 三类干扰

| 类型 | 特征 | 处置 |
|---|---|---|
| **A. 噪声干扰** | 热点 / 碎片文章 / 短视频 / 别人一句话 / 市场风向 | 默认进入观察，不立即行动 |
| **B. 控制干扰** | 外部流程 / 组织压力 / 他人打断 / 项目依赖 / 权限不足 | 转制度接口 / 自动化接口 / 延迟处理 |
| **C. 主观干扰** | 证明欲 / 焦虑换轨 / 新鲜感开坑 / 情绪误判战略 | 24h 冷却 / 72h 微验证 / 禁止开仓 |

**滤波原则**：可能性空间太大时，必须主动滤波。
老茅现在最大的问题不是缺想法，而是**可能性空间太大**。

**落地**：V0.1 = `anti_swing_inbox.md` 处理 C 类；V0.2 = thought_inspector 加 A/B 类分流。

## L3 · 决策层（Frequency Decision）

| 项 | 说明 |
|---|---|
| 核心问题 | 把信号变成可执行的判断 |
| 输入 | L2 分流后的有效信号 |
| 输出 | 六态决策（禁止第七态） |
| 文档来源 | 《全胜手册》三频共振 + 72h MVA |

### 四频评分（V0.2 起在原三频上扩心力频）

| 维度 | 问题 | 1 分 | 5 分 |
|---|---|---|---|
| 理论频谱 | 有没有可复用方法 / 知识树？ | 纯直觉 | 方法论成熟 |
| 市场频谱 | 有没有真实需求 / 定价 / 外部验证？ | 纯想象 | 已有人付费 |
| 组织频谱 | 72h 内能否启动最小行动？ | 完全无准备 | 立即可启动 |
| 心力频谱 | 做完是回血还是耗竭？ | 严重耗竭 | 显著回血 |

评分阈值：
- 总分 ≥ 12 且单维 ≠ 0：可进入主频候选
- 总分 9-12：Watch，30 天后重评
- 总分 < 9：Kill 或噪声观察

### 六态（禁止第七态）

```
All-in       主频开仓，签 90 天锁定
Watch        副频观察，只采样不抢主频
Kill         关停，写入 killed_reasons.md
24h 冷却     摇摆冲动默认处置
72h 验证     战略机会升级路径
入池         普通想法进 thoughts/inbox.md
```

**禁止的第七态**："先放着以后再说。" —— 这就是系统熵增的来源。

**落地**：V0.1 = `goals.md` 锁定协议 + `anti_swing_inbox.md` 升级流程。

## L4 · 执行层（Action Protocol）

| 项 | 说明 |
|---|---|
| 核心问题 | 把判断变成最小动作 |
| 输入 | L3 决策结果 |
| 输出 | 9 类最小动作之一 |
| 文档来源 | 《全胜手册》24h A→D 闭环 |

### 24h A→D 闭环

```
Acquire    采集信号
   ↓
Bridge     桥接到现有项目 / 知识树
   ↓
Construct  构造最小行动
   ↓
Decide     产出明确决断
```

**强制约束**：任何新信息 24h 内必须完成转化，禁止素材囤积。

### 9 类最小动作

```
1. 写一条思考池
2. 建一个 MVA 文件夹
3. 生成一页验证计划
4. 安排 72h 试验
5. 关闭一个噪声源
6. Kill 一个旧项目
7. 补一个复盘
8. 生成一条今日主频建议
9. 升级主频 / 关停主频
```

**原则**：没有动作的信息 = 未闭环信息 = 心力消耗。

**落地**：V0.1 = 老赫在每个采样后主动给出动作建议。

## L5 · 反馈层（Control Feedback）

| 项 | 说明 |
|---|---|
| 核心问题 | 让系统从经验里学习，而不是重复犯错 |
| 输入 | L4 执行结果 + 时间投入数据 |
| 输出 | 五频报告 + 心力账户结算 |
| 文档来源 | 《全胜手册》心力账户 + 《信息传递与控制》闭环 |

### 五频报告

| 频率 | 内容 | 落地 |
|---|---|---|
| **日** | 今日主频 / 投入 / 摇摆 / 噪声 / 心力净额 / 明日咒语 | `daily_log.md` |
| **周** | 主频稳定性 / 摇摆分布 / 噪声 Top 3 / 组织度指数 / 下周决断 | `weekly_report.md` |
| **月** | 经验包袱清零 / 仓位调整 / 信源重评 | `monthly_review.md`（V0.2） |
| **季** | 三频模型校准 / 心力账户季结算 | `quarterly_review.md`（V0.3） |
| **90 天** | 目标锁定协议复核（基于产出数据，不基于情绪） | `goals.md` + `exit_records.md` |

### 心力账户

```
心力净额 = 回血项 - 耗竭项

回血项：
- 主频深度投入（flow 状态）
- 身体能量（睡眠 / 运动 / 饮食）
- 真实外部验证（付费 / 夸奖 / 解决真实痛点）
- 完整时间块

耗竭项：
- 高频噪音时间
- 摇摆冲动登记数
- 工具崇拜 / 计划完美主义 / 复盘沉溺
- 任务切换次数
- 睡眠不足
```

**规则**：心力净额为负 ≥ 3 天 → 系统强制建议进入"低能耗模式"（只做主频，关停所有副频采样）。

**情绪隔离规则**：焦虑度 > 6 禁止新开仓位。

**落地**：V0.1 = `daily_log.md` 第四节心力账户；V0.2 = `frequency_daily.py` 自动计算 7 日均值。

## 控制论公式（来自《信息传递与控制》）

```
现实 (Reality)
    │
    │  L1 采样
    ▼
信号 (Signal)  ────────► L2 滤波 ──► 噪声 / 控制 / 主观
    │
    │  有效信号
    ▼
判断 (Decision) ───────► L3 三频 × 72h MVA ──► 六态
    │
    │  最小动作
    ▼
行动 (Action)  ────────► L4 A→D 闭环 ──► 9 类动作之一
    │
    │  产出 + 数据
    ▼
反馈 (Feedback) ───────► L5 五频报告 ──► 修正下一轮采样
    │
    └────► 缩小可能性空间 ──► 形成控制
```

**核心**：信息 → 控制，靠的是闭环 + 反馈，不是单次决策。

## 不高考只赢原则（来自《全胜手册》）

> 不和别人比同一个赛道，只在自己的频域里取胜。

应用：

1. **不做大众赛道**：不卷通用效率工具 / 通用聊天机器人 / 通用笔记软件
2. **做老茅独有频域**：频域决策 + 核电跨界 + AI 工程 + 虚空建筑师 IP
3. **每个目标必须能回答**：我在哪个频域里能赢？为什么是我？为什么是现在？
4. **如果答不上来 → Kill**，不要硬卷

## 72h MVA 协议（来自《全胜手册》）

```
Day 1  理论验证  ──► 能不能画出知识树？有没有可复用方法？
Day 2  市场验证  ──► 有没有真实需求 / 定价 / 付费意愿证据？
Day 3  组织验证  ──► 72h 内能否启动最小行动？谁来做？

72h 后必须三选一：
  All-in  三频 ≥ 12 且单维 ≠ 0
  Watch   三频 9-12，30 天后重评
  Kill    三频 < 9，写入 killed_reasons.md
```

**禁止**："再观察观察" / "等准备好了再说" / "先放着以后再说"。

---

## 零持仓身份（来自《全胜手册》）

> 72 小时身份模糊化，禁止持有"我必须符合某身份"的固定仓位。

应用：

- 我不是"核电工程师"，也不是"AI 工程师"，也不是"虚空建筑师"
- 我是一个**频域决策者**，根据三频评分动态切换身份
- 身份仓位每月清零一次，不与具体目标绑定
