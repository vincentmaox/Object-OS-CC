# FrequencyOS — Product Requirements Document

## 一句话定位

> ProjectOS 的个人控制论层：把目标、行为、情绪、评价、时间重新统一为一个可采样、可滤波、可决策、可反馈的闭环系统。

## 为什么不是普通效率工具

普通效率工具问：今天有什么任务？
FrequencyOS 问：今天哪些行为真的服从了主频目标？

普通笔记软件保存信息。
FrequencyOS 判断信息能不能转化成行动。

普通 AI 助手迎合用户新想法。
FrequencyOS 反问：你确定这不是高频噪音？

## 设计哲学来源

| 文档 | 提供的核心机制 |
|---|---|
| 《频域组织度重建手册》 | 三频评分 / 90 天锁定 / 摇摆拦截 / 组织度指数 |
| 《信息传递与控制》 | 五层架构 / 噪声滤波 / 控制反馈闭环 / 可能性空间收缩 |
| 《虚空建筑师全胜手册》 | 心力账户 / 不高考只赢 / 72h 微验证 / 零持仓身份 |

三份文档合并到 `control_layer.md` 统一表述。

## V0.1（当前）— Markdown MVP

边界：不写 UI、不接屏幕监控、不接可穿戴、不做心理评估。

5 个闭环：

1. `goals.md` — 目标锁定协议 + 目标池
2. `daily_log.md` — 每日频域采样
3. `anti_swing_inbox.md` — 摇摆冲动缓冲池
4. `weekly_report.md` — 周度频谱报告
5. `prd.md` + `control_layer.md` — 产品定义 + 控制层架构

辅助：

- `templates/` — 四个空白模板
- `scripts/frequency_daily.py` — 独立采样合并入口

V0.1 的成功标准：**连续 30 天每日采样不中断，且组织度指数能稳定 ≥ 70**。

## 不做什么（边界守则）

- **不做打卡 App** — 市场已饱和，且没有方法论锋芒
- **不做心理治疗** — 定位为「目标一致性工具」，避开心理医疗边界
- **不做全自动行为监控** — MVP 不接屏幕 / 可穿戴 / 生理数据
- **不做用户迎合** — 永远先问"这是不是高频噪音"，再决定是否行动
- **不做无限持仓** — 任何想法 / 目标 / 项目都必须进入 A→D 闭环，禁止"先放着以后再说"

## 未来形态：Hermes Reality Sampler

> 把日常行为流转译为项目状态和下一步行动的 AI 现实采样器。
>
> 不是又一个聊天机器人。它把语音、屏幕、文件、飞书、网页、会议，自动采样成：想法、项目、卡点、下一步行动。

### 为什么不是聊天机器人

聊天机器人依赖人主动表达。但高价值信息往往以碎片形式出现：

- 飞书里随手发的一段语音
- 浏览器打开的几个竞品页面
- 本地改了项目但没提交
- 半夜突然想到的产品方向
- 连续三天在同一个项目打转
- 开会反复提到的客户痛点
- 下载了但没读完的 PDF

传统 AI 等你问。现实采样器主动看见。

### 五个采样源

1. **语音** — "我突然想到" → 自动识别归项目
2. **屏幕** — 项目目录 / Git 状态 / 报错反复出现
3. **文件** — README / 设计稿 / PDF / 会议纪要 → 项目资产
4. **沟通** — 飞书消息 / 客户反馈 / 群聊提及 → 市场 / 组织信号
5. **时间** — 项目 72h 无推进 → Watch / Kill 决策区

### MVP 三个闭环

1. **语音想法 → 思考池**（已有，复用 thought_inspector 协议）
2. **项目状态 → 今日建议**（接 ProjectOS registry）
3. **屏幕 / 文件事件 → 自动归类**（资产 → 项目上下文）

### 三层收费设想

- 个人专业版（$10-20/月）— 独立开发者 / 创作者
- 团队版（$20-50/人/月）— 接飞书 / GitHub / Linear / Notion
- 垂直行业版 — 工程审查 / 法律文档 / 咨询交付 / 科研

### 72h MVA 验证方案（待启动时执行）

- Day 1：语音想法入池（目标一天记 10 条真实想法）
- Day 2：项目扫描建议（接 registry，给一句"今日主线"）
- Day 3：卡点提醒（未提交改动 / 缺文档 / 停滞项目）

三个指标：用户是否每天打开 / 是否减少项目焦虑 / 是否推动一个真实闭环。

## 评估节奏

| 频率 | 谁做 | 输出 |
|---|---|---|
| 每日 | 老茅 + 老赫 | `daily_log.md` |
| 每周日 | 老赫 | `weekly_report.md` |
| 每月 | 老茅 | 经验包袱清零 + 仓位调整 |
| 每季 | 老茅 + 老赫 | 三频模型校准 + 信源可信度重评 |
| 每 90 天 | 老茅 | 目标锁定协议复核（基于产出数据，不基于情绪） |
